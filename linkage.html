Compiling and linking a C/C++ application against underling

To compile and link a C or C++ application against your underling installation, the installation's include directory and lib directory must be added to your C/C++ compiler's include and library search paths, respectively. You will need to supply flags to link against underling and FFTW MPI. FFTW and vendor-specific threading flags may also be necessary. Note that you will either need to use your MPI implementation's C/C++ compiler wrapper script, denoted here by $(MPICC)/, or you will additionally need to provide MPI-related flags within the sample commands below.

Assuming you installed pthreads-based threaded FFTW under $FFTWHOME and also assuming at underling configuration time you specified –prefix=$HOME to install underling under your home directory, compiling and linking an MPI application consisting of a single source file named app.c might look as follows:

> $(MPICC) -I$HOME/include app.c -L$HOME/lib -lesio -L$FFTWHOME -lfftw3_mpi -lfftw3_threads -lfftw3 -lm -pthread

This is error prone as you must supply the same FFTW installation directories used at underling configuration time as well as threading details. Continuing the example, if you set your PKG_CONFIG_PATH environment variable to contain $HOME/lib/pkgconfig you can use pkg-config to lookup the relevant compilation and linking information automatically:

> $(MPICC) `pkg-config --cflags underling-fftw-pthreads` app.c `pkg-config --libs underling-fftw-pthreads`

Other pkg-config identifiers you may use include underling, underling-fftw, underling-fftw-pthreads, and underling-fftw-openmp. The first option includes no FFTW dependencies, the second option defaults to using a default threading if it was enabled at FFTW build time or no threading if none was enabled. The third and fourth options select specific threading implementations. The third and fourth options may not be available in FFTW was built without threading.

Using pkg-config will avoid many easy-to-make mistakes. It is highly recommended you use pkg-config with underling where possible.

Using underling with GNU Autoconf/Automake

Because an underling installation provides pkg-config information, using underling with Autoconf and Automake in a C/C++ project is straightforward. The exact details are beyond the scope of this documentation, but the process consists of the following steps:

  1. Use AX_MPI within your configure.ac to determine where your system's MPI compiler wrappers are located.
  2. Ensure that automake is using the MPI compiler wrapper by including a statement like CC=@MPICC@ within your Makefile.am.
  3. Use the PKG_CHECK_MODULES autoconf macro to look for underling's underling-fftw pkg-config information at configure time. Load the information into variables like UNDERLING_CFLAGS and UNDERLING_LIBS.
  4. Add UNDERLING_CFLAGS and UNDERLING_LIBS to the appropriate targets within your Makefile.am.

One very good, but not underling-specific, write up on using GNU Autoconf/Automake with pkg-config is Using C/C++ libraries with Automake and Autoconf.
