Installation

Underling uses the GNU autotools suite (autoconf, automake, and libtool) for its development build system. This system is popular among the Linux development community and provides a familiar build environment for end users.

To build underling starting from a release distribution, untar the distribution and enter the top-level directory.

> tar xvfz underling-$(VERSION).tar.gz
> cd underling-$(VERSION)/

Configuration Requirements

GNU Make
You must have GNU Make installed as underling's build system relies on many GNU extensions.
Compilers
You must have a working C and C++ compiler from the same vendor. Currently the GNU (gcc/g++) and Intel (icc/icpc) compilers are supported. If underling's configure script cannot automatically detect the correct compiler location, try setting $CC and $CXX environment variables prior to running configure.
MPI
You must have a working Message Passing Interface (MPI) implementation. If underling's configure script cannot automatically detect your MPI installation, try setting the $MPICC and $MPIFC environment variables prior to running configure.
FFTW 3.3
You must have a working FFTW 3.3.0 or later installation compiled with MPI parallel support (enabled with –enable-mpi at configure time). You may also wish to enable threading (either or both of –enable-openmp, –enable-threads). The same MPI stack and compiler used to compile FFTW should be used to compile underling.
GRVY (optional)
Underling can optionally use The Groovy Toolkit (GRVY) if it is detected at configure time. If your GRVY installation is not detected correctly, you may use underling's –with-grvy option to specify a location.

To configure underling, within the underling-0.3.2 directory issue

> ./configure

where you should add any additional configure options at the end of the command. One very useful option is the –prefix=DIR which sets your desired top-level installation directory for underling.

Library Build

Once configured, issue a make to build the software. If successful, this will build the underling library (static and dynamic versions), possibly some examples, and some additional underling-related utilities.

> make

Build Verification (Optional)

To optionally verify that the software is working properly, a check target is provided to run a suite of functionality tests against the local build. To run these tests, issue

> make check

Many of the tests require your MPI installation's mpiexec command to be usable. Such tests may not be executable on many public high performance computing platform login nodes.

Also note that the benchmarking tool, underling_bench, has a –check option which can be used to ensure the library is running correctly on any scale problem.

Installation

After the build is complete, issue

> make install

to install the library. The installation will consist of three top-level directories housing underling's include files, compiled libraries, and utilities.
